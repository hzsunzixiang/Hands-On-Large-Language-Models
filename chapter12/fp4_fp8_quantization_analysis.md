# FP4 vs FP8 量化深度分析：QLoRA 分块量化的工程价值

## 零、前置知识：3 条铁律（先记死，后面全靠它们）

### 铁律 1：FP4 原生只能存 0~15 的等级数

FP4（NVFP4）是 4-bit 存储，最多只能表示 2⁴ = **16 个离散的等级编号（0~15）**——永远只有 16 个编号，多一个都没有，这是硬件级硬限制。

**关键认知**：FP4 **不认识** 1.10、1.12、9.15 这些精细浮点数。它只认识 0、1、2、...、15 这 16 个等级编号。你看到的所有精细值（1.10、1.1067、9.15 等），都不是 FP4 直接存储的，而是**等级数通过块的 scale/offset 计算还原**出来的近似值。

### 铁律 2：精细值 = 等级数 × scale + offset（"拼音 + 翻译册"）

FP4 只存"拼音"（等级数 0~15），每个分块配一本"翻译册"（scale + offset），精细值是翻译出来的"汉字"：

```
精细值 = 块最小值(offset) + 等级数 × scale
```

- **同一个等级数 3**，在不同块的翻译册下，还原出完全不同的值：
  - 块 1（1.10~1.20，scale≈0.0067）：等级 3 → 1.10 + 3×0.0067 ≈ **1.1201**
  - 块 2（9.10~9.20，scale≈0.0067）：等级 3 → 9.10 + 3×0.0067 ≈ **9.1201**
- FP4 存的是"拼音 3"，不是"汉字 1.1201"

### 铁律 3：步长(scale)、等级数(16)、覆盖范围此消彼长

```
覆盖范围 = scale ×（等级数 - 1）= scale × 15
```

- scale 越小（翻译册刻度越细）→ 覆盖范围越小 → 还原值越精细
- scale 越大（翻译册刻度越粗）→ 覆盖范围越大 → 还原值越粗糙
- 等级数固定为 16 → **精细度和覆盖范围无法兼得**

> **字典比喻**：
> - FP4 的 16 个等级数（0~15）= 字典里的 16 个"拼音"（固定不变）
> - 每个块的 scale/offset = 一本"专属翻译册"
> - 精细值（1.10、1.12、9.10）= 拼音通过翻译册译出的"汉字"
> - 拼音"3"在块 1 翻译册里 → 汉字"1.1201"
> - 拼音"3"在块 2 翻译册里 → 汉字"9.1201"
> - FP4 只存"拼音"，不存"汉字"；"汉字（精细值）"是拼音 + 翻译册算出来的

## 一、FP4 与 FP8 浮点格式概述

### 1.1 格式定义

| 格式 | 位宽 | 结构 | 精度特征 | 硬件支持 |
|------|------|------|----------|----------|
| **FP8 E4M3** | 8-bit | 1 符号 + 4 指数 + 3 尾数 | 精度 ≈ 0.0625，能区分 1.10/1.12/1.15 等精细值 | Hopper (H100), Blackwell (B200) |
| **FP8 E5M2** | 8-bit | 1 符号 + 5 指数 + 2 尾数 | 更大动态范围，精度略低 | Hopper, Blackwell |
| **FP4 (NVFP4)** | 4-bit | 1 符号 + 2 指数 + 1 尾数 | 仅 16 个离散等级编号（0~15），精细值需通过 scale 还原 | Blackwell (B200, B100) |

### 1.2 性能对比

| 指标 | FP4 (NVFP4) | FP8 | FP16 (基线) |
|------|-------------|-----|-------------|
| 相对吞吐量 (vs H100 FP16) | **3.17×** | 2.80× | 1.0× |
| 模型体积 (相对 FP16) | **3.5× 压缩** | 2× 压缩 | 1.0× |
| FP4 vs FP8 体积比 | **1.8× 更小** | 基线 | — |
| 典型精度损失 | < 1% (配合 two-level scaling) | 极小 | — |
| 主要用途 | **推理** | 训练 + 推理 | 通用 |

### 1.3 NVFP4 的存储结构

```
┌────────────────────────────────────────────────┐
│  16 个 FP4 等级数 (每个 4-bit，值域 0~15)       │  ← 一个 block
│  + 1 个 FP8 E4M3 scale factor (block 级)       │  ← "翻译册"
│  + 1 个 FP32 scale factor (tensor 级，防溢出)   │
└────────────────────────────────────────────────┘
```

> FP4 存的是**等级编号**，不是精细浮点值。精细值在推理时由 `等级数 × block_scale × tensor_scale` 实时还原。

## 二、FP4 的"存储 + 还原"两阶段机制

这是 FP4 量化最关键、也最容易被误解的点。很多人以为 FP4 能直接存储 1.10、1.15 这些值——**不能**，它只存等级编号。

### 2.1 阶段 1：存储（FP8 真值 → FP4 等级数）

以块 1（范围 1.10~1.20，scale≈0.0067）量化 FP8 真值 1.12 为例：

```
第 1 步：计算块内相对位置
  1.12 - 块最小值(1.10) = 0.02

第 2 步：映射到 FP4 等级数
  0.02 / scale(0.0067) ≈ 3（取整）

第 3 步：FP4 最终存储的内容
  → 等级数 3（不是 1.12，也不是 1.10）
```

### 2.2 阶段 2：还原（FP4 等级数 → 近似精细值）

当模型推理需要用这个值时，执行反向计算：

```
第 1 步：读取 FP4 存储的等级数
  → 3

第 2 步：用块的 scale/offset 还原
  近似值 = 块最小值 + 等级数 × scale
         = 1.10 + 3 × 0.0067
         ≈ 1.1201

第 3 步：与原始值对比
  原始 FP8 真值：1.12
  还原近似值：  1.1201
  量化误差：    0.0001（极小）
```

### 2.3 完整等级映射表（块 1：1.10~1.20）

| FP4 存储的等级数 | 还原公式 | 还原出的精细值 |
|:---:|:---|:---:|
| 0 | 1.10 + 0 × 0.0067 | **1.1000** |
| 1 | 1.10 + 1 × 0.0067 | **1.1067** |
| 2 | 1.10 + 2 × 0.0067 | **1.1134** |
| 3 | 1.10 + 3 × 0.0067 | **1.1201** |
| ... | ... | ... |
| 8 | 1.10 + 8 × 0.0067 | **1.1536** |
| ... | ... | ... |
| 15 | 1.10 + 15 × 0.0067 | **1.2005** |

> **核心认知**：右列的"精细值"不是 FP4 直接存储的，它们是**等级数（左列）+ 翻译册（scale=0.0067, offset=1.10）计算还原**出来的近似值。FP4 芯片上真正存的只有左列的 0~15。

### 2.4 为什么不同块的"同一个等级数"还原出完全不同的值？

这就是分块缩放的核心巧思——**不改变 FP4 只能存 0~15 的硬件限制，而是给每个块配不同的翻译册**：

| 块 | 范围 | scale | 等级数 3 的还原值 |
|:---|:-----|:------|:---:|
| 块 1 | 1.10~1.20 | 0.0067 | 1.10 + 3×0.0067 ≈ **1.12** |
| 块 2 | 9.10~9.20 | 0.0067 | 9.10 + 3×0.0067 ≈ **9.12** |
| 块 3 | 100.0~110.0 | 0.6667 | 100.0 + 3×0.6667 ≈ **102.0** |

> 等级数 3 是同一个"拼音"，但在三本不同的"翻译册"下，翻译出了 1.12、9.12、102.0 三个完全不同的"汉字"。

## 三、量化实验：FP8 → FP4 的真实误差分析

### 3.1 场景设定

模型某层权重（FP8 E4M3 可精确表示的 7 个值）：

```
[1.10, 1.12, 1.15, 1.20,  9.10, 9.15, 9.20]
```

核心矛盾：
- FP8 E4M3 有 3-bit 尾数，**能直接精确表示 1.12**
- FP4 NVFP4 只存 0~15 的等级数，**1.12 需要通过等级数 + scale 还原**，存在量化误差

### 3.2 全局量化：灾难性信息丢失

全局范围 `[1.10, 9.20]`，跨度 = 8.1

16 个等级数要覆盖 8.1 的范围，scale = 8.1 / 15 ≈ **0.54**

每个等级之间的还原值间隔高达 0.54，绝大部分精细信息丢失：

```
存储过程（FP8 真值 → FP4 等级数）：
  1.10：(1.10-1.10)/0.54 = 0   → FP4 存等级 0  → 还原 1.10+0×0.54 = 1.10
  1.12：(1.12-1.10)/0.54 ≈ 0   → FP4 存等级 0  → 还原 1.10+0×0.54 = 1.10
  1.15：(1.15-1.10)/0.54 ≈ 0   → FP4 存等级 0  → 还原 1.10+0×0.54 = 1.10
  1.20：(1.20-1.10)/0.54 ≈ 0   → FP4 存等级 0  → 还原 1.10+0×0.54 = 1.10
  9.10：(9.10-1.10)/0.54 ≈ 15  → FP4 存等级 15 → 还原 1.10+15×0.54 = 9.20
  9.15：(9.15-1.10)/0.54 ≈ 15  → FP4 存等级 15 → 还原 1.10+15×0.54 = 9.20
  9.20：(9.20-1.10)/0.54 = 15  → FP4 存等级 15 → 还原 1.10+15×0.54 = 9.20
```

**结果：7 个不同的 FP8 值 → FP4 只用了 2 个等级数（0 和 15）→ 还原后只剩 2 个值，信息损失 > 90%**

根本原因：scale 太大（0.54），16 个等级的间隔太粗，1.10/1.12/1.15/1.20 全部被映射到同一个等级数 0，无法区分。

### 3.3 QLoRA 分块量化：误差压缩到可容忍范围

将权重按局部性切成两块，**每块独立配一本"翻译册"（scale/offset）**——本质是让每个块的 scale 极小，16 个等级数能区分出极精细的差异：

#### 块 1：`[1.10, 1.12, 1.15, 1.20]`

- 局部范围：`[1.10, 1.20]`，跨度仅 0.1
- scale = 0.1 / 15 ≈ **0.0067**（比全局 scale 精细 80 倍）

```
存储 + 还原过程：
  1.10：(1.10-1.10)/0.0067 = 0  → 存等级 0  → 还原 1.10+0×0.0067 = 1.1000  ✅ 误差 0
  1.12：(1.12-1.10)/0.0067 ≈ 3  → 存等级 3  → 还原 1.10+3×0.0067 = 1.1201  ≈ 误差 0.0001
  1.15：(1.15-1.10)/0.0067 ≈ 7  → 存等级 7  → 还原 1.10+7×0.0067 = 1.1469  ≈ 误差 0.003
  1.20：(1.20-1.10)/0.0067 = 15 → 存等级 15 → 还原 1.10+15×0.0067 = 1.2005 ≈ 误差 0.0005
```

> **关键观察**：FP4 并没有直接存储 1.12 这个值。它存的是等级数 3，还原后得到 1.1201（≈1.12）。误差从全局量化的"完全无法区分"变成了 0.0001 级别——这就是分块缩放的威力。

#### 块 2：`[9.10, 9.15, 9.20]`

- 局部范围：`[9.10, 9.20]`，跨度仅 0.1
- 同理，scale≈0.0067，16 个等级聚焦在极小区间

```
存储 + 还原过程：
  9.10：存等级 0  → 还原 9.10+0×0.0067 = 9.1000  ✅
  9.15：存等级 7  → 还原 9.10+7×0.0067 = 9.1469  ≈ 误差 0.003
  9.20：存等级 15 → 还原 9.10+15×0.0067 = 9.2005 ≈ 误差 0.0005
```

### 3.4 核心对比

| 量化方式 | FP4 实际存储 | 还原结果 | 与 FP8 对比 | 精度损失 |
|---------|------------|---------|------------|---------|
| **全局 FP4** | `[0,0,0,0,15,15,15]`（等级数） | `[1.10, 1.10, 1.10, 1.10, 9.20, 9.20, 9.20]` | 7 值 → 2 值 | **> 90%** |
| **QLoRA 分块 FP4** | 块1:`[0,3,7,15]` 块2:`[0,7,15]`（等级数） | `[1.10, 1.12, 1.15, 1.20, 9.10, 9.15, 9.20]` | 几乎完美还原 | **< 1%** |

> 注意：QLoRA 的"1.12"实际是还原近似值 1.1201，不是 FP4 直接存储的。但误差极小，工程上可忽略。

## 四、QLoRA 分块量化的本质

### 4.1 它做了什么

```
           全局量化                           分块量化
  ┌────────────────────┐        ┌──────────┐  ┌──────────┐
  │ 1 本翻译册(scale大) │        │ 翻译册 A  │  │ 翻译册 B  │
  │ 覆盖 1.10 ─── 9.20 │        │ 1.10─1.20│  │ 9.10─9.20│
  │ scale ≈ 0.54       │        │scale≈0.007│  │scale≈0.007│
  │ 等级间隔太粗        │        │等级间隔精细│  │等级间隔精细│
  └────────────────────┘        └──────────┘  └──────────┘
         粗糙                       精细           精细
```

**核心动作**：不改变 FP4"只能存 0~15"的硬件限制，而是把一本覆盖范围巨大的"粗糙翻译册"（全局 scale），**拆分成多本覆盖范围极小的"精细翻译册"（块级 scale）**。

同一个"拼音（等级数）"，通过精细翻译册，就能还原出足够精确的"汉字（精细值）"。

### 4.2 它没做什么

- **没有**改变 FP4 只能存 0~15 等级数的硬件限制
- **没有**让 FP4 直接存储 1.12 这个浮点数
- **没有**创造 FP4 原本不存在的表达能力
- 你看到的精细值（1.10、1.12、9.15）都是**还原出来的近似值**，不是 FP4 原生能表达的
- 量化误差**依然存在**，只是从"不可接受"变成了"模型可容忍"

### 4.3 数学本质

```
全局：  scale = (global_max - global_min) / 15 = 8.1 / 15 ≈ 0.54
分块：  scale = (block_max  - block_min)  / 15 = 0.1 / 15 ≈ 0.007
```

scale 压缩比 = 0.54 / 0.007 ≈ **80×**

还原精度提升的本质：**缩小每本翻译册的覆盖范围，让同样 16 个等级数对应的刻度间隔更细**。不是"给 FP4 增加了格子"，而是"让每个格子翻译出的值更精确"。

## 五、硬件落地意义

在 NVIDIA Blackwell GPU (B200/B100) 上，QLoRA + NVFP4 实现：

| 收益 | 数值 |
|------|------|
| 模型体积缩小 | **1.8×**（vs FP8），**3.5×**（vs FP16） |
| 推理吞吐量提升 | **3.17×**（vs H100 FP16） |
| 整体精度损失 | **< 1%** |

这就是低比特量化能真正落地 LLM 大规模部署的核心原因：
**不是消除误差，而是通过精细的翻译册（block scale），把等级数的还原误差控制在模型鲁棒性的容忍范围内，同时换取数倍的效率提升。**

## 六、个人理解与补充

### 6.1 为什么 QLoRA 用分块而不是更复杂的方法？

分块量化的精妙在于其**工程简洁性**：
- 不需要重新训练模型（Post-Training Quantization）
- 不需要复杂的校准数据集
- 只需要一个简单的设计决策：**block size 选多大？**（通常 64 或 128 个权重为一块）
- 额外存储开销极小：每块只多存一个 FP8 scale factor（即一本"翻译册"的代价）

这符合工程中的帕累托法则——用 20% 的复杂度解决 80% 的精度问题。

### 6.2 分块量化的局限性

1. **块内异常值（outlier）问题**：如果一个块内出现一个极端值（如 `[1.10, 1.12, 1.15, 100.0]`），该块的 scale 会被拉大（scale = (100.0-1.10)/15 ≈ 6.6），16 个等级的间隔变粗，1.10/1.12/1.15 又回到无法区分的状态。这也是为什么 LLM.int8() 等方法会对 outlier 做特殊处理（mixed-precision decomposition）。

2. **block size 的权衡**：
   - 块越小 → 每块覆盖范围越小 → scale 越细 → 还原精度越高，但翻译册（scale factor）的存储开销越大
   - 块越大 → 存储效率越高，但块内值域范围可能很大 → scale 变粗 → 退化为"全局量化"
   - NVFP4 选择 16 元素为一块，是 NVIDIA 在硬件 Tensor Core 对齐、存储开销、精度之间的平衡点

3. **FP4 的 16 个等级是物理极限**：无论怎么分块、怎么调 scale，FP4 在每个块内也只有 16 个可区分的等级。当块内权重分布高度非均匀时（如集中在某个小区间），有些值必然会发生碰撞（多个不同原始值映射到同一个等级数）。

### 6.3 从信息论角度看

FP8 → FP4 本质是将每个权重的信息量从 8 bit 压缩到 4 bit，**信息熵减半**。分块量化的 scale factor 相当于额外编码了"这个块的翻译册长什么样"的**元信息**（值域在哪里、刻度间隔多大），用少量额外 bit 换回了大量还原精度，是一种高效的信息编码策略。

### 6.4 与 GGUF 量化的关联

本文档讨论的 NVFP4 是 NVIDIA GPU 上的硬件原生格式。而在 CPU/Apple Silicon 上广泛使用的 GGUF 格式（如 `Qwen3.5-27B-Q4_K_M.gguf`）采用的是类似思想但不同实现的 **K-Quant 分块量化**：

- Q4_K_M 中的 "K" = K-Quant（分块量化）
- "M" = Medium（精度和体积的中等平衡）
- 同样采用 block-wise scaling，每 32 或 256 个权重为一个 super-block
- 区别在于 K-Quant 使用整数量化而非浮点量化，且 block 结构更复杂（有 sub-block 嵌套）

**本质相同**：都是通过为每个块配备专属的"翻译册"（scale/offset），让固定数量的等级编号能还原出不同区间的精细值——硬件不同，思想一致。
