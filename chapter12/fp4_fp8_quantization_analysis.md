# FP4 vs FP8 量化深度分析：QLoRA 分块量化的工程价值

## 零、前置知识：3 条铁律（先记死，后面全靠它们）

### 铁律 1：FP4 的格子数固定为 16

FP4（NVFP4）是 4-bit 存储，最多只能表示 2⁴ = **16 个离散值（量化等级）**——永远只有 16 个"格子"，多一个都没有，这是硬件级硬限制。

### 铁律 2：步长、格子数、覆盖范围的此消彼长

```
覆盖范围 = 步长 ×（格子数 - 1）= 步长 × 15
```

- 步长越细 → 覆盖范围越小
- 步长越粗 → 覆盖范围越大
- 格子数固定为 16 → **精细度和覆盖范围无法兼得**

### 铁律 3：16 个格子由整数部分和小数部分共享

FP4 的 16 个格子是**全局统一分配**的，要同时覆盖「整数部分」和「小数部分」：

- 整数部分量级越大 → 占用格子越多 → 小数部分分到的格子越少 → **小数精度越差**
- 整数部分量级越小 → 占用格子越少 → 小数部分分到的格子越多 → **小数精度越好**

> **钱包比喻**：把 16 个格子看作"16 个硬币"，要同时买"整数面包"和"小数咸菜"：
> - 买 1 元的面包 → 花 1 个硬币 → 剩 15 个硬币挑咸菜 → 精细到 0.11
> - 买 10000 元的面包 → 花 15 个硬币 → 只剩 1 个硬币 → 只能选 0.10 或 0.12，买不到 0.11

## 一、FP4 与 FP8 浮点格式概述

### 1.1 格式定义

| 格式 | 位宽 | 结构 | 精度特征 | 硬件支持 |
|------|------|------|----------|----------|
| **FP8 E4M3** | 8-bit | 1 符号 + 4 指数 + 3 尾数 | 精度 ≈ 0.0625，能区分 1.10/1.12/1.15 等精细值 | Hopper (H100), Blackwell (B200) |
| **FP8 E5M2** | 8-bit | 1 符号 + 5 指数 + 2 尾数 | 更大动态范围，精度略低 | Hopper, Blackwell |
| **FP4 (NVFP4)** | 4-bit | 1 符号 + 2 指数 + 1 尾数 | 仅 16 个离散等级，最小可区分间隔 ≥ 0.5 | Blackwell (B200, B100) |

### 1.2 性能对比

| 指标 | FP4 (NVFP4) | FP8 | FP16 (基线) |
|------|-------------|-----|-------------|
| 相对吞吐量 (vs H100 FP16) | **3.17×** | 2.80× | 1.0× |
| 模型体积 (相对 FP16) | **3.5× 压缩** | 2× 压缩 | 1.0× |
| FP4 vs FP8 体积比 | **1.8× 更小** | 基线 | — |
| 典型精度损失 | < 1% (配合 two-level scaling) | 极小 | — |
| 主要用途 | **推理** | 训练 + 推理 | 通用 |

### 1.3 NVFP4 的存储结构

```
┌────────────────────────────────────────────────┐
│  16 个 FP4 权重值 (每个 4-bit)                    │  ← 一个 block
│  + 1 个 FP8 E4M3 scale factor (block 级)         │
│  + 1 个 FP32 scale factor (tensor 级，防溢出)     │
└────────────────────────────────────────────────┘
```

## 二、"16 个格子"的直觉：为什么相同区间长度，精度天差地别？

这是 FP4 量化最关键、也最容易被忽略的点。用两个极端例子一看就懂。

### 2.1 场景对比：`1.10~1.12` vs `10000.10~10000.12`

两个区间的共同点：
- 区间长度**完全相同**：都是 0.02
- FP4 格子总数**完全相同**：都是 16

唯一区别：**整数部分的量级**（1 vs 10000）

### 2.2 场景 A：量化 `1.10 ~ 1.12`（小整数，格子几乎全给小数）

**步长** = 0.02 / 15 ≈ 0.001333

**格子分配**：
- 整数部分"1"：只占 **1 个格子**
- 小数部分 0.10~0.12：分到 **15 个格子**
- 小数格子宽度 ≈ 0.001333

```
量化 1.11：
  落在 1.10 + 8×0.001333 ≈ 1.1107 这个格子
  → 舍入误差仅 0.0007
  ✅ 小数精度极高
```

### 2.3 场景 B：量化 `10000.10 ~ 10000.12`（巨大整数，格子几乎全被整数吃掉）

16 个格子要先"够到"10000 这个整数：
- 整数部分"10000"：需要占用 **15 个格子**
- 小数部分 0.10~0.12：只剩 **1 个格子**
- 小数格子宽度 = 0.02 / 1 = 0.02

```
量化 10000.11：
  只有「10000.10」和「10000.12」两个刻度
  10000.11 被舍入到 10000.10 或 10000.12
  → 误差 0.01，是场景 A 的 14 倍
  ❌ 小数精度极差
```

### 2.4 格子分配图（核心！）

| 区间 | 总格子 | 整数占用 | 小数占用 | 小数步长 | 量化误差 |
|------|--------|---------|---------|---------|---------|
| `1.10~1.12` | 16 | **1** | **15** | ≈0.001333 | ≈0.0007 |
| `10000.10~10000.12` | 16 | **15** | **1** | 0.02 | 0.01 |

> **结论**：两个区间长度完全相同（0.02），但因为整数部分量级不同，16 个格子在「整数/小数」间的分配比例**完全颠倒**，小数精度差 14 倍。

### 2.5 反证：强行用细步长量化大整数会怎样？

假设不管整数量级，硬要用场景 A 的细步长（0.001333）去覆盖 `10000.10~10000.12`：

```
覆盖 10000 需要的格子 = 10000 / 0.001333 ≈ 750 万个
FP4 只有 16 个格子 → 完全不够

结果：10000.11 直接溢出，被截断成 FP4 能表示的最大值
     → 整数部分错得离谱，小数精度完全丢失
```

**这就解释了为什么不能无脑追求细步长——格子不够连整数都放不下。**

## 三、量化实验：FP8 → FP4 的真实误差分析

### 3.1 场景设定

模型某层权重（FP8 E4M3 可精确表示的 7 个值）：

```
[1.10, 1.12, 1.15, 1.20,  9.10, 9.15, 9.20]
```

核心矛盾：
- FP8 E4M3 有 3-bit 尾数，**能精确表示 1.12**
- FP4 NVFP4 仅 1-bit 尾数，**天然无法表示 1.12**

### 3.2 全局量化：灾难性信息丢失

全局范围 `[1.10, 9.20]`，跨度 = 8.1

FP4 仅 16 个格子，步长 = 8.1 / 15 ≈ **0.54**

用第二章的"格子分配"视角理解：16 个格子要同时覆盖从 1 到 9 的整数跨度，小数部分几乎分不到格子。

```
原始值    →  量化值    误差
─────────────────────────
1.10     →  1.0      0.10
1.12     →  1.0      0.12   ← 误差显著
1.15     →  1.0      0.15
1.20     →  1.0      0.20
9.10     →  9.0      0.10
9.15     →  9.0      0.15
9.20     →  9.0      0.20
```

**结果：7 个不同的 FP8 值 → 只剩 2 个 FP4 值，信息损失 > 90%**

### 3.3 QLoRA 分块量化：误差压缩到可容忍范围

将权重按局部性切成两块，**每块独立计算 scale factor**——本质是**让每个块的 16 个格子只服务于极小的局部范围**：

#### 块 1：`[1.10, 1.12, 1.15, 1.20]`

- 局部范围：`[1.10, 1.20]`，跨度仅 0.1
- 整数部分只有"1"，占 1 个格子，剩下 15 个格子全给小数
- 块内步长 = 0.1 / 15 ≈ **0.0067**（比全局精度提升 80 倍）

```
1.10  →  1.10   (正好落在刻度上)
1.12  →  1.10   (FP4 无 1.12，舍入到最近可表示值，误差 0.02)
1.15  →  1.15   (正好落在刻度上)
1.20  →  1.20   (正好落在刻度上)
```

> **关键观察**：1.12 → 1.10，这是"FP8 能精确表示，但 FP4 无论如何表示不了"的**真实、不可消除的量化误差**。QLoRA 没有凭空创造 FP4 不存在的表达能力，只是把误差从 0.12 压缩到了 0.02。

#### 块 2：`[9.10, 9.15, 9.20]`

- 局部范围：`[9.10, 9.20]`，跨度仅 0.1
- 同理，16 个格子聚焦在极小区间，精度极高

```
9.10  →  9.10
9.15  →  9.15
9.20  →  9.20
```

### 3.4 核心对比

| 量化方式 | 结果 | 与 FP8 对比 | 精度损失 |
|---------|------|------------|---------|
| **全局 FP4** | `[1.0, 1.0, 1.0, 1.0, 9.0, 9.0, 9.0]` | 7 值 → 2 值 | **> 90%** |
| **QLoRA 分块 FP4** | `[1.10, 1.10, 1.15, 1.20, 9.10, 9.15, 9.20]` | 仅 1.12 舍入为 1.10 | **< 5%** |

## 四、QLoRA 分块量化的本质

### 4.1 它做了什么

```
             全局量化                         分块量化
     ┌──────────────────┐          ┌─────────┐  ┌─────────┐
     │  16 个格子覆盖    │          │ 16 个格子 │  │ 16 个格子 │
     │  1.0 ──── 9.2    │          │ 1.10─1.20│  │ 9.10─9.20│
     │  步长 ≈ 0.54     │          │步长≈0.007 │  │步长≈0.007 │
     │  整数吃掉大量格子  │          │整数仅占1格│  │整数仅占1格│
     └──────────────────┘          └─────────┘  └─────────┘
           粗糙                        精细           精细
```

**核心动作**：将 FP4 有限的 16 个格子，从"覆盖巨大全局范围（整数吃掉大量格子）"**聚焦到极小的局部块内（整数只占极少格子，小数精度最大化）**。

用"钱包分币"的话说：QLoRA 把"买 1 万元面包的账单"和"买 1 元面包的账单"**分开结算**，每张账单各分 16 个硬币，这样买 1 元面包时，15 个硬币都能花在咸菜（小数精度）上。

### 4.2 它没做什么

- **没有**让 FP4 凭空表示出 1.12
- **没有**创造 FP4 原本不存在的表达能力
- **没有**改变"16 个格子"的硬件限制
- 量化误差**依然存在**，只是从"不可接受"变成了"模型可容忍"

### 4.3 数学本质

```
全局：  误差 ∝ (global_max - global_min) / 2^bits = 8.1 / 16 ≈ 0.54
分块：  误差 ∝ (block_max  - block_min)  / 2^bits = 0.1 / 16 ≈ 0.007
```

误差压缩比 = 0.54 / 0.007 ≈ **80×**

本质上，分块量化通过**按整数量级分块**，优化了 16 个格子在"整数/小数"间的分配效率：
- 全局量化：格子要跨越 1~9 的整数，小数精度被牺牲
- 分块量化：每个块的整数范围极小，格子几乎全部留给小数

## 五、硬件落地意义

在 NVIDIA Blackwell GPU (B200/B100) 上，QLoRA + NVFP4 实现：

| 收益 | 数值 |
|------|------|
| 模型体积缩小 | **1.8×**（vs FP8），**3.5×**（vs FP16） |
| 推理吞吐量提升 | **3.17×**（vs H100 FP16） |
| 整体精度损失 | **< 1%** |

这就是低比特量化能真正落地 LLM 大规模部署的核心原因：
**不是消除误差，而是把误差控制在模型鲁棒性的容忍范围内，同时换取数倍的效率提升。**

## 六、个人理解与补充

### 6.1 为什么 QLoRA 用分块而不是更复杂的方法？

分块量化的精妙在于其**工程简洁性**：
- 不需要重新训练模型（Post-Training Quantization）
- 不需要复杂的校准数据集
- 只需要一个简单的设计决策：**block size 选多大？**（通常 64 或 128 个权重为一块）
- 额外存储开销极小：每块只多存一个 FP8 scale factor

这符合工程中的帕累托法则——用 20% 的复杂度解决 80% 的精度问题。

### 6.2 分块量化的局限性

1. **块内异常值（outlier）问题**：如果一个块内出现一个极端值（如 `[1.10, 1.12, 1.15, 100.0]`），该块的 scale factor 会被拉大——用"格子分配"视角看，就是整数部分"100"突然吃掉大量格子，小数精度急剧退化。这也是为什么 LLM.int8() 等方法会对 outlier 做特殊处理（mixed-precision decomposition）。

2. **block size 的权衡**：
   - 块越小 → 精度越高（每个块的整数范围越小，格子分配越合理），但 scale factor 的存储开销越大
   - 块越大 → 存储效率越高，但越可能包含跨量级的值（大整数和小整数混在一块），退化为"全局量化"
   - NVFP4 选择 16 元素为一块，是 NVIDIA 在硬件 Tensor Core 对齐、存储开销、精度之间的平衡点

3. **FP4 的 1-bit 尾数是物理极限**：无论怎么分块，FP4 在每个块内也只有 16 个可表示值。当块内权重分布高度非均匀时（如集中在某个小区间），有些值必然会发生碰撞（多个不同原始值映射到同一个 FP4 值）。

### 6.3 从信息论角度看

FP8 → FP4 本质是将每个权重的信息量从 8 bit 压缩到 4 bit，**信息熵减半**。分块量化的 scale factor 相当于额外编码了"这个块的值域在哪里"的**元信息**（即"这个块的整数量级是多少"），用少量额外 bit 换回了大量精度，是一种高效的信息编码策略。

### 6.4 与 GGUF 量化的关联

本文档讨论的 NVFP4 是 NVIDIA GPU 上的硬件原生格式。而在 CPU/Apple Silicon 上广泛使用的 GGUF 格式（如 `Qwen3.5-27B-Q4_K_M.gguf`）采用的是类似思想但不同实现的 **K-Quant 分块量化**：

- Q4_K_M 中的 "K" = K-Quant（分块量化）
- "M" = Medium（精度和体积的中等平衡）
- 同样采用 block-wise scaling，每 32 或 256 个权重为一个 super-block
- 区别在于 K-Quant 使用整数量化而非浮点量化，且 block 结构更复杂（有 sub-block 嵌套）

**本质相同**：都是通过缩小量化范围来提升低比特格式的有效精度——让 16 个格子不用浪费在跨越巨大整数量级上，而是专注服务于局部小范围内的精细区分。
